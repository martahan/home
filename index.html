A prototype framework under ongoing development that allows developers to analyse the performance of their Python programs by writing queries and analysing the results.

Actively developed at CERN, at the CMS Experiment, and The University of Manchester.

Author: Joshua Dawes


Using VyPR is easy (to make a start, see here):

1 - Querying

You use VyPR's library to write a query about the performance of a part of your program.

Queries written for VyPR take the form of properties that a run of that part of the program should hold.

2 - Instrumentation

VyPR determines where in your program to take data to check your query.

3 - Monitoring

At runtime, VyPR resolves your query by taking measurements.

Analysing Query Results

VyPR collects detailed (but conservative) information at runtime that lets you see:

Whether the properties your queries defined were satisfied.

Whatever the outcome, which parts of the code were responsible.

Given a group of query results, whether surrounding code may be responsible.

See How VyPR Works

We provide a browser-based tool to visualise VyPR instrumenting and then monitoring a program.

You can set up the tool using

curl -L http://cern.ch/vypr/setup-vypr-visualisation -o setup-vypr-visualisation && source setup-vypr-visualisation && launch-vypr-vis

and then use the tool by accessing here.
